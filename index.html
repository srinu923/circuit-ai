The Problem

Why Does This Matter?

Power electronics engineers spend 2–3 hours per circuit manually transcribing schematics into LTspice netlists — with a 15–20% error rate. Hand-drawn sketches from brainstorming sessions and IC datasheet schematics sit completely disconnected from simulation tools.

Existing EDA tools like KiCad and Eagle only work with clean, digitally-drawn schematics. OCR tools extract text but have zero understanding of circuit topology. No open tool bridges the gap from a raw schematic image to a validated, runnable simulation.

Inputs

Two Types of Schematics

The pipeline handles two very different input types with dedicated agents for each.

Pipeline A — Hand-Drawn

Pipeline B — App Note

Architecture

Multi-Agent Pipeline

Five cooperative AI agents work in sequence. GPT-4.1-mini replaces CNN-based symbol classifiers, Hough transform wire tracers, and OCR extractors — all through two specialized structured JSON prompts.

01

Image Preprocessing

Gaussian blur (σ=1.5) + Otsu thresholding enhances contrast and suppresses noise before sending to the vision model.

preprocess.py
02

Vision Parser Agent

GPT-4.1-mini with VISION_SYSTEM_PROMPT extracts components, wires, junctions and texts as strict JSON. Temperature=0.1 for deterministic output.

vision_parser.py GPT-4.1-mini
03

Graph Builder Agent

Constructs electrical net graph using pin-snap geometry (radius=18px). Detects floating pins, disconnected nets, and missing ground references.

graph_builder.py
04

Topology Repair Agent

Domain-aware rule-based rewiring for common power electronics topologies. Each repair action is logged with reason and confidence score (0.97 for all 4 repairs).

topology_repair.py
05

IC Template Matcher (App Note only)

Fuzzy part-number matching + pin label overlap scoring. LT1374HV matched at 0.90 confidence. Score = 0.75 × NameSim + 0.25 × PinOverlap.

ic_template_matcher.py
06

Netlist Generator + Validator

Converts repaired nets to LTspice .cir syntax. Validates syntax, .model definitions, and node assignments. Generates confidence score.

netlist_generator.py validator.py
07

LTspice Runner + Waveform Viewer

Python subprocess invokes LTspice 26.0.1 in batch mode. Parses .raw output and plots waveforms via matplotlib. Target nodes selectable via CLI or natural language prompt.

ltspice_runner.py
Output

Auto-Generated Netlists

Both pipelines produce LTspice-compatible .cir files. Note the correct peak voltage conversion: 220V RMS → 311.127V peak (220√2).

// Hand-drawn: Half-wave rectifier

* Auto-generated from hand-drawn schematic MVP VAC1 N001 0 SINE(0 311.127 50) D1 N001 N002 D1 L1 N002 N003 1mH C1 N003 0 10uF R1 N003 0 100 .tran 0 100m .model D1 D .end

// App note: LT1374HV Buck Converter

* LT1374HV Buck Converter (auto-generated from app-note schematic) V1 IN 0 DC 12 C3 IN 0 10u ; Input cap C2 N001 N002 0.27u ; Bootstrap cap L1 N002 OUT 5u ; Inductor R1 OUT N004 5.36k ; Feedback top R2 0 N004 4.99k ; Feedback bot C1 OUT 0 100u ; Output cap XU1 IN N001 N004 0 OUT N005 N003 N002 LT1374 top=10 bot=1T .lib LT1374.sub .tran 10m startup .end
